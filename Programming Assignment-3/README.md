# Programming Assignment 3 — COVID-19 Contact Tracing

**Course:** COP 3502C
**Author:** Ryan Ramdihal
**Language:** C

## Overview

This program simulates a basic contact-tracing tool. Given the user's location on a Cartesian plane and the coordinates of people infected with COVID-19, it sorts the infected points by distance from the user (closest first) and then answers a series of location-based queries to determine whether an infected person is at a given point and, if so, their rank in the sorted list.

## Algorithms Used

The program is built around three required algorithms:

- **Merge Sort** — used as the primary recursive sorting algorithm for larger subarrays.
- **Insertion Sort** — used on subarrays whose size is less than or equal to a threshold value `t`. This hybrid approach reduces overhead from recursion on small partitions.
- **Binary Search** — used after sorting to look up query points efficiently in O(log n) time.

The sort wrapper function (`sort`) is non-recursive and decides whether to call `insertSort` directly or hand the array off to the recursive `mergeSort`, which further dispatches to insertion sort whenever a subsection's size falls at or below the threshold.

## Input Format (`assignment3input.txt`)

The first line contains five space-separated integers:

```
x y n s t
```

Where:

- `x y` — the user's location (|x|, |y| ≤ 10,000)
- `n` — number of infected people (2 ≤ n ≤ 10^6)
- `s` — number of query points (1 ≤ s ≤ 2 × 10^5)
- `t` — threshold for switching between Merge Sort and Insertion Sort (1 ≤ t ≤ 30)

The next `n` lines contain the coordinates of infected people. The following `s` lines contain query points.

## Output Format (`out.txt`)

The first `n` lines list the infected coordinates sorted by:

1. Ascending distance from the user
2. Ascending x-coordinate (for ties in distance)
3. Ascending y-coordinate (for ties in both distance and x)

The last `s` lines report each query in one of two formats:

```
x y found at rank R
x y not found
```

`R` is the 1-based rank of the matching infected person in the sorted list.

## Compilation & Execution

Because the program uses `<math.h>`, compile with the `-lm` flag:

```bash
gcc assignment3-1.c -o assignment3 -lm
./assignment3
```

The program reads `assignment3input.txt` from the current working directory and writes results to `out.txt` in the same directory.

## Implementation Notes

- The user's location is stored in a single global struct (`location`); no other globals are used, per the assignment restrictions.
- Coordinates are stored in a `CoordStruct` containing two integer fields (`x`, `y`).
- The `compareTo` function compares two points by Euclidean distance from the user, breaking ties first on x and then on y, returning a negative value, zero, or a positive value following standard comparator conventions.
- `ReadData` handles reading the `n` infected coordinates from the input file into the array.
- `mergeSort` is the recursive routine that either recurses further or delegates to `insertSort` once the partition size is within the threshold.
- `binSearch` reuses `compareTo` to navigate the sorted array, returning the matching index or `-1` if the point is not found.
